========================================
EXPERIMENT ANALYSIS REPORT
BraTS 3D Brain Tumor Segmentation
Experiments 1 - 5
========================================
Date: April 2026
Task: Multi-class 3D tumor segmentation on BraTS-GLI dataset
Classes: Background (BG), Tumor Core (TC), Edema (ED), Enhancing Tumor (ET)
Input: 4-channel MRI (T1, T1ce, T2, FLAIR) | Output: 4-class segmentation mask
Hardware: NVIDIA GeForce RTX 4080 | Framework: PyTorch 2.5.1+cu121

========================================
EXPERIMENT GROUP OVERVIEW
========================================

Group 1 - Weak Baseline (Experiments 1 & 2):
  Exp 1: UNet3D          (poor preprocessing, no tumor sampling)
  Exp 2: Attention UNet  (poor preprocessing, no tumor sampling)

Group 2 - Improved Pipeline (Experiments 3, 4, 5):
  Exp 3: UNet3D          (proper pipeline, improved sampling)
  Exp 4: Attention UNet  (proper pipeline, improved sampling)
  Exp 5: UNet++          (proper pipeline, improved sampling)

Trainer version: segmentation_v0 (all 5 experiments)
Loss function:   Dice + Cross-Entropy (unweighted)
Optimizer:       Adam, lr=0.0001 (constant)
Epochs:          100 (all experiments)

========================================
EXPERIMENT 1 & 2 — WEAK BASELINE
========================================

Configuration:
  - patch_size:       [64, 64, 64]
  - No proper Z-score normalization
  - No tumor-focused patch sampling
  - Full volume or naive patch loading
  - No sliding window inference at test time

Observations:
  - Extremely poor Dice scores across all tumor classes
  - Model collapsed to predicting background (BG ~1.0, TC/ED/ET ~0.0)
  - No stable convergence observed
  - High instability in prediction maps

Root Cause Analysis:
  1. Small patch size misses context needed for tumor localization
  2. Without normalization MRI intensity distributions are model-breaking
  3. No tumor-focused sampling means patches are almost entirely background
  4. Without sliding window inference, boundary artifacts dominate at test time
  5. Model optimizes CE on background-dominated volumes → trivial solution

Key Insight:
  Data preprocessing is the primary bottleneck, not model architecture.
  Both UNet and Attention UNet failed identically, confirming the pipeline
  is broken, not the models.

Status: INVALID BASELINES
  These experiments are not comparable to the literature.
  Retained only to document the necessity of proper preprocessing.

========================================
EXPERIMENT 3 — UNet3D (IMPROVED PIPELINE)
========================================

Configuration changes from Exp 1:
  - patch_size:         [128, 128, 128]
  - Z-score normalization applied per brain foreground mask
  - Tumor-aware patch sampling (tumor_ratio=0.5)
  - Sliding window inference with 50% overlap + Gaussian weighting
  - Loss: Dice + CE (unweighted)
  - Epochs: 100, Adam lr=0.0001

Results:
  Mean Dice:            0.8062
  Dice per class:
    BG (Background):    0.9991
    TC (Tumor Core):    0.5630   <-- weakest class
    ED (Edema):         0.8540
    ET (Enhancing):     0.8086
  HD95:                 6.986
  Note: Sensitivity/Specificity reported as >2.0 — metric computation
        bug detected (class-sum instead of class-mean); values excluded.

Analysis:
  + Major performance jump from Exp 1/2 — pipeline is now valid
  + Strong segmentation of Edema (0.854) and Enhancing Tumor (0.809)
  + Sliding window with Gaussian weighting removes patch boundary artifacts
  - Tumor Core (TC) stuck at 0.563 — consistent bottleneck
  - TC is the smallest and most heterogeneous class; class imbalance dominates

Conclusion:
  Pipeline validated. Strong baseline established.
  Architecture is not the bottleneck — TC is limited by training strategy.

========================================
EXPERIMENT 4 — Attention UNet (IMPROVED PIPELINE)
========================================

Configuration:
  Same as Exp 3 with Attention UNet architecture.
  Attention gates added at skip connections to spatially reweight features.

Results:
  Mean Dice:            0.8054
  Dice per class:
    BG (Background):    0.9991
    TC (Tumor Core):    0.5630   <-- same as UNet3D
    ED (Edema):         0.8415   (slightly lower than UNet3D)
    ET (Enhancing):     0.8181   (slightly higher than UNet3D)
  HD95:                 5.605    (improvement over UNet3D: 6.986 → 5.605)

Analysis:
  + HD95 improved by 1.38 units — attention helps boundary precision
  + Slightly better Enhancing Tumor segmentation
  - Mean Dice marginally lower than plain UNet3D (0.8054 vs 0.8062)
  - TC unchanged at 0.563 — attention does not resolve class imbalance
  - Attention adds parameters but does not improve the hardest class

Key Insight:
  Attention gates improve spatial selectivity and boundary quality (HD95)
  but have no effect on class imbalance, which remains the true limiter.
  Architecture complexity ≠ class imbalance solution.

Conclusion:
  Attention UNet is a marginal improvement in boundary quality only.
  Does not surpass UNet3D in mean Dice.
  Architecture is confirmed NOT to be the current bottleneck.

========================================
EXPERIMENT 5 — UNet++ (IMPROVED PIPELINE)
========================================

Configuration:
  Same improved pipeline as Exp 3/4 with UNet++ architecture.
  Dense skip connections: nested, cascaded multi-scale pathways.
  Depth: 2 levels (shallower than UNet3D/Attention UNet to match capacity).

Results:
  Mean Dice:            0.7956   <-- WORSE than UNet3D and Attention UNet
  Dice per class:
    BG (Background):    0.9991
    TC (Tumor Core):    0.5630   (same bottleneck)
    ED (Edema):         0.8412
    ET (Enhancing):     0.7790   <-- lowest ET among all three models
  HD95:                 5.309    (slightly better than Exp 3, worse than Exp 4)

Analysis:
  - Mean Dice dropped from 0.806 (UNet3D) to 0.796
  - ET segmentation noticeably degraded (0.779 vs 0.809/0.818)
  - TC unchanged at 0.563 — confirms this is a training issue, not architecture
  + HD95 slightly better than UNet3D
  - Dense skip connections increase parameter count without sufficient training

Root Causes of Degraded Performance:
  1. Undertraining: same 100 epochs insufficient for higher model complexity
  2. No deep supervision: UNet++ designed to benefit from auxiliary losses
     at intermediate nodes — without this, dense paths provide noise
  3. Shallow depth (2 levels) limits multi-scale feature extraction
  4. No learning rate scheduling: flat lr unable to escape local minima
     at higher complexity

Key Insight:
  UNet++ requires: (a) deep supervision, (b) longer training,
  (c) learning rate scheduling. Without these, complex architecture
  becomes a liability, not an advantage.
  The problem is training strategy, not model capacity.

Conclusion:
  UNet++ is not properly utilized here.
  These results do not represent UNet++'s true capability.
  Must be revisited with appropriate training setup.

========================================
CROSS-EXPERIMENT INSIGHTS
========================================

1. DATA PIPELINE IS THE FOUNDATION
   Without proper normalization, patch size, and tumor-aware sampling,
   no architecture can learn meaningful tumor representations.
   The jump from Exp 1-2 to Exp 3-5 is entirely due to pipeline fixes,
   not model changes.

2. CLASS IMBALANCE IS THE PRIMARY BOTTLENECK (Phase 1 Conclusion)
   TC Dice is locked at 0.5630 across UNet3D, Attention UNet, and UNet++.
   This uniformity across architectures conclusively shows the issue is:
   - Loss function not penalizing TC misclassification enough
   - Background over-represented even with tumor_ratio=0.5
   - Unweighted CE treats all voxels equally — catastrophic for tiny TC region

3. ATTENTION HELPS BOUNDARIES, NOT CLASS BALANCE
   Attention UNet: HD95 5.60 vs UNet3D HD95 6.99 (20% improvement)
   But TC Dice is identical (0.563). Spatial attention improves
   boundary delineation but cannot compensate for loss imbalance.

4. COMPLEX MODELS NEED MATCHING TRAINING STRATEGY
   UNet++ with naive training (100 epochs, flat lr, unweighted loss)
   performs worse than a simpler UNet with the same setup.
   Model complexity without training strategy alignment is harmful.

5. METRIC BUG NOTE
   Sensitivity and specificity in Exp 3-5 are summed across classes
   rather than averaged (values > 2.0 confirm this). These metrics
   are excluded from analysis. Fixed in v1 evaluation.

========================================
WHAT WAS LEARNT — ACTIONABLE LESSONS
========================================

Lesson 1: Fix class imbalance before changing architecture
  → Must apply weighted loss (TC weight ≥ 3.0) in next phase

Lesson 2: Tumor sampling ratio must be aggressive
  → tumor_ratio from 0.5 to 0.8 ensures TC appears in nearly every patch

Lesson 3: Longer training with LR decay is essential for complex models
  → UNet++ needs 200 epochs with cosine annealing (not constant LR)

Lesson 4: Focal loss helps hard voxel learning
  → Focal term with gamma=2 down-weights easy background voxels,
     concentrating gradients on difficult tumor boundaries

Lesson 5: Architecture comparison is only valid after equalizing training quality
  → Current comparison (Exp 3 vs 4 vs 5) is not fair for UNet++

========================================
DRAWBACKS OF EXPERIMENTS 1-5
========================================

1. Experiments 1-2 produced no usable metrics — pipeline was fundamentally broken
2. Experiments 3-5 use unweighted loss — TC is systematically neglected
3. All experiments use constant learning rate — suboptimal convergence
4. UNet++ not given sufficient epochs for its capacity
5. No deep supervision for UNet++
6. Metric computation bug in sensitivity/specificity (v0 evaluation)
7. tumor_ratio=0.5 insufficient for TC-focused learning
8. No mixed precision training in v0 trainer (slower iteration)

========================================
NEXT STEPS — ADDRESSED IN EXPERIMENTS 6-8
========================================

The following changes were applied in the v1 training phase:

1. Loss function redesign
   - Weighted Dice Loss: class weights [0.1, 3.0, 1.0, 2.0] (BG, TC, ED, ET)
   - Weighted Cross-Entropy with same weights
   - Focal Loss (gamma=2.0, focal_weight=0.5) for hard voxel focus
   - Combined: Loss = WeightedDice + WeightedCE + 0.5 * FocalLoss

2. Training configuration upgrades
   - Epochs: 100 → 150 (UNet3D, Attention UNet), 200 (UNet++)
   - LR Scheduler: CosineAnnealingLR (lr: 1e-4 → 1e-6)
   - Mixed Precision: AMP enabled (GradScaler)

3. Sampling upgrade
   - tumor_ratio: 0.5 → 0.8

4. Trainer upgrade: segmentation_v0 → segmentation_v1
   - Per-epoch logging of dice/ce/focal loss components separately
   - Scheduler integration

Stop condition target set at: Mean Dice ≥ 0.86, TC ≥ 0.70
