========================================
EXPERIMENT ANALYSIS REPORT
BraTS 3D Tumor Segmentation
========================================

----------------------------------------
📌 EXPERIMENT GROUP OVERVIEW
----------------------------------------

Total Experiments: 5

Group 1 (Baseline - Weak Setup):
- Exp 1: UNet (poor preprocessing)
- Exp 2: Attention UNet (poor preprocessing)

Group 2 (Improved Pipeline):
- Exp 3: UNet (improved preprocessing + sampling)
- Exp 4: Attention UNet (improved preprocessing)
- Exp 5: UNet++ (improved preprocessing)

----------------------------------------
🧪 EXPERIMENT 1 & 2 (WEAK BASELINE)
----------------------------------------

Setup:
- No proper normalization
- No tumor-focused sampling
- Basic loading (full volume / naive patches)
- No sliding window inference

Observations:
- Very poor Dice scores
- Model biased towards background
- Tumor regions poorly segmented
- High instability in predictions

Key Insight:
- Data preprocessing is CRITICAL
- Model architecture had negligible effect
- Pipeline was the bottleneck, not the model

Conclusion:
❌ These experiments are INVALID baselines
✔ Used only to validate pipeline importance

----------------------------------------
🧪 EXPERIMENT 3 (UNet - IMPROVED)
----------------------------------------

Scores:
- Dice: ~0.806
- TC: ~0.56
- ED: ~0.85
- ET: ~0.81
- HD95: ~6.98

Improvements Applied:
- Z-score normalization
- Patch-based sampling
- Tumor-aware sampling
- Sliding window inference

Insights:
✔ Huge jump in performance
✔ Stable training behavior
✔ Good segmentation of large regions (ED)

Weakness:
❌ Tumor Core (TC) remains low (~0.56)

Conclusion:
✔ Strong baseline established
✔ Pipeline validated successfully

----------------------------------------
🧪 EXPERIMENT 4 (ATTENTION UNET)
----------------------------------------

Scores:
- Dice: ~0.805
- TC: ~0.56
- ED: ~0.84
- ET: ~0.82
- HD95: ~5.60

Insights:
✔ Slight improvement in boundary quality (HD95 ↓)
✔ Better ET segmentation

Observation:
❌ No improvement in TC
❌ Overall performance ~same as UNet

Key Insight:
→ Attention improves spatial focus
→ Does NOT solve class imbalance

Conclusion:
✔ Attention UNet is NOT significantly better
✔ Architecture is NOT current bottleneck

----------------------------------------
🧪 EXPERIMENT 5 (UNET++)
----------------------------------------

Scores:
- Dice: ~0.795 ❌
- TC: ~0.56
- ED: ~0.84
- ET: ~0.77
- HD95: ~5.30

Observations:
❌ Performance dropped vs UNet
✔ Slightly better HD95

Root Causes:
1. Undertraining (same epochs, higher complexity)
2. No deep supervision (critical for UNet++)
3. Higher parameter count → needs more data
4. Training instability

Key Insight:
→ Complex model without proper training = worse results

Conclusion:
❌ UNet++ not properly utilized
✔ Not a model issue, but training strategy issue

----------------------------------------
📊 CROSS-EXPERIMENT INSIGHTS
----------------------------------------

1. DATA PIPELINE > MODEL ARCHITECTURE
------------------------------------
- Biggest gain came from preprocessing, not model
- Proper normalization + patching is mandatory

2. CLASS IMBALANCE IS MAIN BOTTLENECK
------------------------------------
- Tumor Core consistently ~0.56 across all models
- Indicates training limitation, not architecture

3. ATTENTION ≠ PERFORMANCE BOOST
------------------------------------
- Helps boundaries (HD95)
- Does not improve hardest class (TC)

4. COMPLEX MODELS NEED STRONG TRAINING
------------------------------------
- UNet++ underperformed due to:
  - insufficient epochs
  - lack of deep supervision
  - no loss optimization

5. CURRENT LIMITATION
------------------------------------
→ Training strategy is limiting performance
→ Not model capacity

PHASE 1: TRAINING OPTIMIZATION (HIGH PRIORITY)
----------------------------------------------

1. Weighted Loss (MANDATORY)
   - Add class weights to CE
   - Focus on Tumor Core

2. Advanced Loss
   - Dice + Focal Loss
   - Improve hard voxel learning

3. Increase Tumor Sampling
   - tumor_ratio → 0.8+

4. Increase Sliding Window Overlap
   - stride → patch_size / 4

5. Train Longer
   - epochs → 150–200

STOP CONDITION:
- Mean Dice ≥ 0.83
- TC ≥ 0.65
